# Vision

## Why this project exists

Life Tracker is not a productivity application.

It is a Personal Operating System.

The goal is not to collect data.

The goal is to improve decision making.

Every interaction should help the user understand:

- what happened
- why it happened
- what should happen next

The application should gradually become a trusted second brain.

---

## Core Principles

### Reduce Cognitive Load

The application should remove mental overhead.

Users should never wonder:

- Where do I put this?
- What should I do next?
- Which page should I open?

The application should answer those questions naturally.

---

### Encourage Reflection

Tracking without reflection creates noise.

Reflection transforms experiences into learning.

Every module should eventually answer:

"What have I learned?"

instead of only

"What happened?"

---

### Long-term Thinking

Optimize for years.

Not days.

Features that become annoying after one month should not exist.

---

### Calm by Default

The interface should feel peaceful.

Avoid:

- dashboards everywhere
- gamification
- excessive colors
- unnecessary animations
- vanity metrics

Prefer whitespace.

Prefer typography.

Prefer clarity.

---

### Quality over Quantity

Adding ten mediocre features is worse than improving one excellent feature.

Always ask:

Does this improve the user's life?

If not, don't build it.

---

### Human First

The product exists for one human.

Not investors.

Not marketing.

Not enterprise customers.

Every feature should feel personal.

---

## Success

The application succeeds when the user:

- makes better decisions
- remembers more
- reflects more deeply
- worries less
- grows consistently

instead of simply filling more data.
