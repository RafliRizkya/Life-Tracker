# Design System

Design is not decoration.

Design communicates meaning.

---

# Personality

The interface should feel:

- calm
- premium
- editorial
- thoughtful
- warm

Never feel:

- corporate
- noisy
- gamified
- childish

---

# Typography

Typography creates hierarchy.

Use typography before using color.

Whitespace before borders.

Weight before decoration.

---

# Spacing

Prefer generous spacing.

Dense interfaces increase cognitive load.

Every section should breathe.

---

# Cards

Cards should group related information.

Do not create cards only for decoration.

Avoid nested cards.

---

# Colors

Use color intentionally.

Accent colors indicate importance.

Not decoration.

Neutral colors should dominate.

---

# Visual Hierarchy

Users should know:

1. where to look
2. what matters
3. what to do next

without thinking.

---

# Icons

Icons support labels.

Icons never replace labels.

Prefer Lucide consistency.

---

# Motion

Animation explains.

Animation should never distract.

Use motion for:

- hierarchy
- continuity
- feedback

Never animate simply because it looks nice.

---

# Empty States

Every empty state should teach.

Examples:

No finance records

↓

Start recording your first expense.

Instead of

↓

No data.

---

# Error States

Errors should explain:

- what happened
- why
- what to do next

---

# Loading

Loading should reduce uncertainty.

Prefer skeletons.

Avoid long blank screens.

---

# Mobile

Mobile is the primary platform.

Desktop should feel like an enhancement,

not the default.
