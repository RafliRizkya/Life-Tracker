# Architecture Review

Use before making architectural changes.

---

## Goal

Improve architecture without unnecessary complexity.

---

## Analyze

Understand:

- current architecture
- data flow
- dependencies
- state management
- reusable code

---

## Questions

What problem exists?

Is architecture actually the problem?

Can the issue be solved with smaller changes?

Would new abstractions increase complexity?

---

## Deliverables

Explain:

Current architecture

Problems

Options

Tradeoffs

Recommendation

Migration plan

Regression risks

---

## Rules

Never redesign because something looks old.

Architecture changes must produce measurable benefits.

Prefer evolutionary improvements.

Avoid revolutionary rewrites.
