# Refactoring

Goal:

Improve code without changing behavior.

---

## Before Refactoring

Understand:

- purpose
- dependencies
- public API
- existing behavior

---

## Improve

Reduce duplication.

Improve naming.

Extract reusable logic.

Remove dead code.

Simplify conditions.

Reduce complexity.

---

## Preserve

Behavior.

API.

UX.

Performance.

Accessibility.

---

## Avoid

Large rewrites.

Style-only changes.

Moving files without reason.

Changing architecture unnecessarily.

---

## Validation

Existing functionality must continue working.

No regressions.

No duplicated logic.

---

## Deliverables

Explain:

What changed.

Why.

Expected maintenance improvements.
