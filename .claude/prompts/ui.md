# UI & UX Design

Use this prompt whenever creating or modifying the user interface.

---

## Goal

Design interfaces that reduce cognitive load.

The interface should disappear.

The user's attention should stay on their life,
not on the application.

---

## Think Like

- Apple Journal
- Linear
- Notion
- Reflect
- Readwise Reader

Do NOT copy their appearance.

Understand their principles.

---

## Before Designing

Understand:

- user goal
- emotional state
- primary action
- frequency of use
- mobile experience

Ask:

"What is the easiest possible experience?"

---

## Visual Hierarchy

Every screen should have:

1. One primary action
2. One clear focal point
3. Predictable navigation
4. Logical spacing

Avoid competing elements.

---

## Typography

Typography should create hierarchy.

Use:

- size
- weight
- spacing

before adding colors.

---

## Cards

Cards exist to organize information.

Avoid decorative cards.

Avoid card inside card inside card.

---

## Color

Color communicates meaning.

Use accent colors sparingly.

Neutral colors should dominate.

---

## Motion

Motion should explain.

Use animation to:

- reveal hierarchy
- communicate state
- reinforce continuity

Never animate purely for aesthetics.

Respect prefers-reduced-motion.

---

## Empty States

Every empty state should:

- teach
- encourage
- explain the next action

Avoid:

"No data."

Prefer:

"Start by adding your first transaction."

---

## Loading

Prefer skeletons.

Avoid blank pages.

Avoid layout shift.

---

## Accessibility

Keyboard navigation.

Touch target >= 44px.

WCAG AA.

Visible focus states.

Readable typography.

---

## Responsive

Design mobile first.

390px is the reference width.

Desktop should enhance the experience.

Never sacrifice mobile quality.

---

## Deliverables

Explain:

- UX rationale
- hierarchy decisions
- interaction flow
- responsive considerations
