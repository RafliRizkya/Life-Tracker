# Feature Implementation

Use this prompt whenever implementing a new feature.

---

## Objective

Implement the feature while preserving:

- architecture
- UX
- maintainability
- performance

---

## Before Coding

Understand:

- user problem
- affected modules
- existing implementation
- reusable components
- existing utilities

Never duplicate logic.

---

## During Implementation

Prefer:

- extending existing components
- reusing utilities
- incremental changes

Avoid:

- rewriting unrelated code
- introducing unnecessary patterns
- large refactors

---

## UX

Every feature should:

- reduce friction
- feel natural
- support keyboard navigation
- work on mobile

---

## Code Quality

Prefer readable code.

Prefer explicit naming.

Keep functions focused.

Keep components reasonably sized.

Remove dead code.

---

## Validation

Verify:

- functionality
- responsive layout
- accessibility
- console errors
- edge cases

---

## Final Summary

Explain:

- what changed
- why
- affected modules
- future improvements

Generate a commit message.

Do not commit automatically.
