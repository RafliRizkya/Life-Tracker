# Frontend Checklist

## User Experience

- [ ] User goal is obvious within 3 seconds.
- [ ] One primary CTA per screen.
- [ ] No unnecessary clicks.
- [ ] Empty state exists.
- [ ] Loading state exists.
- [ ] Error state exists.
- [ ] Success feedback exists.
- [ ] Forms feel conversational.
- [ ] Mobile experience feels natural.
- [ ] Navigation is predictable.

---

## UI

- [ ] Consistent spacing.
- [ ] Typography hierarchy is clear.
- [ ] Colors follow design system.
- [ ] No visual clutter.
- [ ] Icons are meaningful.
- [ ] Cards are not overused.
- [ ] Alignment is consistent.
- [ ] No overflowing elements.

---

## Accessibility

- [ ] Keyboard navigation works.
- [ ] Focus visible.
- [ ] Screen reader labels.
- [ ] Touch targets >=44px.
- [ ] Contrast WCAG AA.
- [ ] prefers-reduced-motion respected.

---

## Performance

- [ ] No unnecessary rerenders.
- [ ] Lazy loading considered.
- [ ] Images optimized.
- [ ] Bundle impact reasonable.

---

## Quality

- [ ] Console clean.
- [ ] No TODO left.
- [ ] No debug logs.
- [ ] TypeScript passes.
- [ ] Lint passes.

---

## Validation

- [ ] Playwright executed.
- [ ] Mobile tested.
- [ ] Desktop tested.
